# Glosario de Términos Técnicos

Definiciones de términos utilizados en la documentación de NEOPapaya.

## A
* **ALSA**: Arquitectura de sonido de Linux. Usada para acceso directo al hardware.
* **AppArmor**: Sistema de seguridad para restringir capacidades de programas.

## C
* **CI/CD**: Integración y Despliegue Continuo.
* **Context Window**: Límite de tokens que el LLM puede procesar.

## E
* **Edge Computing**: Procesamiento local de datos para reducir latencia y mejorar privacidad.
* **Embedding**: Representación vectorial de texto/audio.

## F
* **Fail2Ban**: Prevención de intrusiones mediante análisis de logs.
* **Fine-Tuning**: Reentrenamiento de un modelo base con datos específicos.

## G
* **GGUF**: Formato unificado para modelos cuantizados (llama.cpp).
* **Guardrails**: Mecanismos de seguridad en IA.

## H
* **Hallucination**: Cuando la IA inventa datos incorrectos.

## I
* **IaC**: Infraestructura como Código (Docker, K8s).
* **Intent**: La intención del usuario (NLU).

## L
* **Latency**: Retardo en la respuesta.
* **LLM**: Large Language Model (Gemma, Llama).
* **LoRA**: Adaptación de bajo rango para fine-tuning eficiente.

## M
* **MQTT**: Protocolo de mensajería ligero IoT.

## N
* **NLU**: Comprensión del Lenguaje Natural.

## O
* **OOM Killer**: Mecanismo del kernel para liberar RAM matando procesos.

## Q
* **Quantization**: Reducción de precisión (bits) del modelo para ahorrar memoria.

## R
* **RAG**: Generación Aumentada por Recuperación (Búsqueda en docs locales).

## S
* **STT**: Speech-to-Text (Voz a Texto).
* **System Prompt**: Instrucción inicial que define la personalidad de la IA.

## T
* **Temperature**: Creatividad del modelo (0.0 - 1.0).
* **Token**: Unidad de texto procesada por la IA.
* **TTS**: Text-to-Speech (Texto a Voz).

## V
* **VAD**: Detección de Actividad de Voz.
* **Vector Database**: Base de datos para búsquedas semánticas (ChromaDB).

## W
* **Wake Word**: Palabra de activación ("Colega").
* **WAL**: Write-Ahead Logging (SQLite).
